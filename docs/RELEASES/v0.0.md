# FrilVault v0.0 Release History

The v0.0 series represents the initial public preview of FrilVault.

These releases may contain incomplete features, packaging limitations,
and breaking changes.

---

## v0.0.2

### New Features

- Added platform-specific VSIX packages for:
  - `darwin-arm64`
  - `darwin-x64`
  - `linux-x64`
  - `win32-x64`

### Improvements

- Bundled the FrilVault CLI with the VS Code extension.
- Removed the requirement to install `flvt` separately for supported VS Code installs.
- Improved release delivery by attaching platform-specific VSIX packages to each GitHub Release.

### Bug Fixes

- Fixed note creation failing on fresh Marketplace installations when no external `flvt` binary was installed.
- Improved diagnostics when the bundled CLI cannot start or does not match the expected version.

### Other Changes

- `frilvault.cliPath` is now an optional advanced override.
- The extension uses its bundled CLI by default.
- Marketplace publishing and GitHub Release asset generation now run as separate workflows.

### Install

- Install from the Visual Studio Marketplace for the default experience.
- Install from GitHub Release by downloading the matching VSIX and using `Extensions: Install from VSIX...` in VS Code.
- Marketplace users do not need to choose a platform manually. The Marketplace selects the matching package automatically.

### Upgrade Notes

No vault data migration is required.

---

## v0.0.1

### Initial Public Preview

- Published the first FrilVault VS Code extension.
- Added line and symbol anchored notes.
- Added local `.vault` persistence.
- Added note creation, editing, deletion, search, statistics, health, and repair features.

### Known Issue

The extension required a separately installed `flvt` CLI.

Without the CLI, note persistence failed with a generic `Save failed`
message. This limitation was fixed in v0.0.2.
